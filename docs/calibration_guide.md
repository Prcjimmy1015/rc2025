## 概述

本文档描述5舵机D1机械臂在比赛场地上的完整标定流程。

**机械臂舵机定义**：

| 舵机ID | 名称 | 功能 |
|--------|------|------|
| 0 | 基座旋转 | 控制机械臂水平旋转方向 |
| 1 | 大臂俯仰 | 控制手臂高度和前伸距离 |
| 2 | 小臂俯仰 | 辅助控制前伸距离 |
| 3 | 腕部俯仰 | 控制末端（夹爪）角度 |
| 4 | 夹爪 | 500=张开，2000=闭合 |

**角度范围**: 1000-2000，对应 0-240°（约 1 单位 ≈ 0.24°）

**依赖文件**:
- `d1_arm/scripts/d1_arm.py` → `D1Arm` 类
- `d1_arm/scripts/d1_pick.py` → 交互测试脚本（可参考修改）

**标定代码位置**: `d1_arm/scripts/arm_actions.py` → `Poses` 类 和 `Calibration` 类

## 一、机械臂姿态标定

### 准备工作

1. 四足机器人停放在比赛场地对应平台旁边
2. 机械臂通过串口连接到机器人上的计算平台（`/dev/ttyUSB0`）
3. 编写或使用已有的交互测试脚本，逐舵机调整角度

### 操作方法

以 `d1_pick.py` 为模板，编写一个标定用脚本 `calibrate_pose.py`：

```python
#!/usr/bin/env python3
import sys, os, time
sys.path.insert(0, os.path.dirname(os.path.abspath(__file__)))
from d1_arm import D1Arm

arm = D1Arm(port="/dev/ttyUSB0", baudrate=1000000)

# 逐舵机测试
# arm.write_angle(舵机ID, 角度, 运动时间ms)
arm.multi_write([1500, 2000, 1500, 1500, 500], 1000)
```

逐姿态调整5个舵机角度值，直到满足要求。每种姿态记录一组 `[base, arm, forearm, wrist, gripper]` 角度值。

## 二、像素→世界坐标标定

### 原理

通过3组对应的（像素坐标，世界坐标）点对，使用 OpenCV `cv2.getAffineTransform()` 计算 2×3 仿射变换矩阵，将 D435 图像中检测到的像素坐标转换为机械臂末端的世界坐标。

**坐标系统约定**:
- X: 左右方向（机械臂基座坐标系）
- Y: 高度（由 D435 深度值 mm 提供）
- Z: 前后方向

### 步骤

#### 步骤1: 准备标定物

使用一个直径约 3cm 的红色小球或任何在 D435 图像中清晰可见的小物体。

#### 步骤2: 调整到拍照姿态

```python
arm.multi_write(Poses.PHOTO, 1000)
time.sleep(1.5)
```

#### 步骤3: 采集3组标定点

将标定物放在平台上3个不同位置，位置尽量分散（覆盖平台整个区域）。

对每个位置:

**A. 记录像素坐标**
```python
# 拍照并保存
color_frame, depth_frame = camera.get_aligned_frames()
color_image = camera.get_color_image(color_frame)
cv2.imwrite(f"calib_point_{i}.png", color_image)
```
在PC上打开保存的图像，用画图/PhotoShop 查看鼠标坐标，记录标定物中心的 (x, y) 像素值。

**B. 记录世界坐标**

手动测量：移动机械臂末端（夹爪中心）到标定物正上方，刚好接触标定物表面。读取舵机角度后计算末端的世界坐标位置。

测量方法：
```python
for sid in range(5):
    angle, t = arm.read_angle(sid)
    print(f"舵机{sid}: {angle}")
```

根据 5 个舵机角度和机械臂几何模型（URDF 中定义），通过正运动学计算末端位置 (X, Y, Z)。在标定阶段，为简化操作，也可以用直尺手动测量末端相对于机械臂基座的物理位置。

#### 步骤4: 填入标定数据

将3组数据填入 `d1_arm/scripts/arm_actions.py` 的 `Calibration.__init__()` 中：

```python
self.pixel_points = np.float32([
    [px1, py1],   # 点1的像素坐标
    [px2, py2],   # 点2的像素坐标
    [px3, py3],   # 点3的像素坐标
])
self.world_points = np.float32([
    [wz1, wx1],   # 点1的世界坐标 (Z, X)
    [wz2, wx2],   # 点2的世界坐标 (Z, X)
    [wz3, wx3],   # 点3的世界坐标 (Z, X)
])
```

#### 步骤5: 验证标定

将标定物放在一个新位置，拍照识别像素坐标 → 通过仿射矩阵计算世界坐标 → 手动移动机械臂到计算坐标，对比误差。

## 三、舵机→世界坐标映射标定

`arm_actions.py` 中有3个占位函数需要实测校准：

### 3.1 `_calc_base_from_wx(wx)` — 世界X → 基座角度

**目的**: 给定左右偏移量 (wx, mm)，计算基座舵机(舵机0)的目标角度。

**方法**:
1. 将标定物放在平台左侧边缘，记录世界X坐标 `wx_left`，手动调整基座使夹爪对准→记录基座角度 `base_left`
2. 将标定物放在平台右侧边缘，记录世界X坐标 `wx_right`，手动调整基座使夹爪对准→记录基座角度 `base_right`

| 位置 | 世界X (mm) | 基座角度 |
|------|:----------:|:-------:|
| 最左 | `______` | `______` |
| 中间 | 0 | 1500 |
| 最右 | `______` | `______` |

计算斜率: `slope = (base_right - base_left) / (wx_right - wx_left)`

修改代码中的公式: `base = 1500 + wx * slope`

### 3.2 `_calc_arm_from_wz(wz)` — 世界Z → 大臂角度

**目的**: 给定前后距离 (wz, mm)，计算大臂舵机(舵机1)的目标角度。

**方法**:
1. 标定物放在平台最近处，记录世界Z坐标 `wz_near` → 手动调大臂对准 → 记录 `arm_near`
2. 标定物放在平台最远处，记录世界Z坐标 `wz_far` → 手动调大臂对准 → 记录 `arm_far`

## 附录: 常见问题

**Q: 夹爪闭合后物资掉落？**
A: 调整 `GRIPPER_CLOSE` 值（默认2000），增大夹持力。注意过大会损坏舵机。

**Q: 抓取时碰到平台表面？**
A: 增大 `_calc_grip_arm_from_depth()` 的输出角度，或调整 Pre-Pick 姿态抬高起始位置。

**Q: 识别成功率低？**
A: 检查光照条件、D435相机对焦、YOLO模型是否用实际比赛物资训练过。
